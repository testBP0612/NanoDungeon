# 02 — GAME DESIGN 遊戲設計

本檔定義玩法規則，是實作的規格來源（source of truth）。數值放在 `Data/*.json`，本檔只描述規則與關係。

---

## 核心循環（Core Loop）

```txt
開始戰鬥
  ↓
本回合獲得 N 顆球（N 由玩家屬性決定，初始見 04_BALANCE_RULES）
  ↓
玩家逐顆瞄準並發射（一次一顆）
  ↓
球碰撞釘子，累積本回合傷害與效果
  ↓
所有球落底（或超時回收）
  ↓
一次結算總傷害給敵人
  ↓
┌─ 敵人死亡？
│    是 → 進入升級三選一 → 下一場戰鬥
│    否 → 敵人攻擊玩家（扣 HP）→ 回到「本回合獲得 N 顆球」
  ↓
玩家 HP 歸零 → 死亡結算（Game Over）
擊敗 Boss → 勝利結算（Victory）
```

## 戰鬥流程（Battle Flow，回合制）

一場戰鬥由多個**回合（Round）**組成，直到敵人死亡或玩家死亡：

1. **回合開始**：玩家獲得 N 顆球（球種組成見 Ball 系統）。回合傷害累計器歸零、回合倍率重置。
2. **發射階段**：玩家逐顆瞄準發射。球受重力與彈跳物理影響，撞擊釘子觸發效果，把傷害加進「回合傷害累計器」。
3. **落底**：每顆球碰到底部或超時即回收。所有球回收後進入結算。
4. **結算階段**：把「回合傷害累計器」一次套用到敵人 HP。此時才結算落底類效果（如 Blast Ball 加成）。
5. **敵人回合**：若敵人存活，敵人對玩家造成攻擊傷害（受護盾類效果影響）。
6. **判定**：玩家 HP ≤ 0 → 死亡結算；敵人 HP ≤ 0 → 若為 Boss 則勝利結算，否則進入升級三選一後打下一場。

> 設計重點：傷害是**回合一次結算**，不是每次命中即時扣血。這讓「堆疊本回合傷害」成為核心策略。

## Peg 系統（釘子）

場上散布發光能量節點（Peg）。球命中時觸發效果並累積傷害。MVP 僅 4 種，資料見 `Data/pegs.json`。

| Peg | 行為 | effect_type |
| --- | --- | --- |
| Normal Peg 普通釘 | 命中造成基礎傷害 | `damage` |
| Burst Peg 爆裂釘 | 命中造成較高傷害 | `damage` |
| Heal Peg 補血釘 | 命中回復少量玩家 HP | `heal` |
| Double Peg 倍傷釘 | 命中後提升本回合傷害倍率（限次，見 Q-001） | `damage_multiplier` |

規則：
- 命中傷害先乘上「當前回合倍率」再累積。
- Heal Peg 的回血可即時或於結算時套用（實作建議結算時統一處理，見 `03_SYSTEM_SPEC.md`）。
- Double Peg 觸發次數受限（暫行：每回合 1 次、倍率 ×2，見 `OPEN_QUESTIONS.md` Q-001）。
- **再命中冷卻（re-hit cooldown）**：同一顆球命中同一釘後，於冷卻時間內的重複接觸不計分（預設 0.2 秒，數值見 feel 設定），避免「卡釘」farming（見 Q-010）。不同球、或冷卻後的命中仍正常計分。
- **不實作連鎖釘**（避免鄰近判定與遞迴觸發風險）。
- **不實作釘子耗損 / 消失**（屬 MVP 外的玩法擴張，見 Q-010 選項 B）。

## Ball 系統（球種）

玩家每回合的球由不同球種組成。MVP 僅 3 種，資料見 `Data/balls.json`。

| Ball | 行為 | effect_type |
| --- | --- | --- |
| Normal Ball 普通球 | 基礎能量球，無額外效果 | `none` |
| Blast Ball 爆破球 | 落底時額外加成一次「本回合最高單次釘子傷害」（見 Q-002） | `on_drop_bonus` |
| Shield Ball 護盾球 | 本回合敵人攻擊傷害降低（見 Q-003） | `damage_reduction` |

規則：
- 初始只給 Normal Ball；Blast / Shield 透過升級或精英怪解鎖。
- 球種效果分兩類：**命中時**（影響傷害累積）與**落底 / 結算時**（影響最終結算與下一次敵人攻擊）。
- Blast Ball 的落底加成**可隨球數疊加**：每顆 Blast 球各加成一次「本回合最高單次傷害」（見 Q-012；Q-002 仍採含倍傷後數值）。
- Shield Ball 減傷**不疊加**：多顆取最高（見 Q-003）。
- **不實作連射球**（避免節奏失控與物理狀態複雜化）。

## Enemy 系統（敵人）

MVP 為固定 5 場戰鬥，資料見 `Data/enemies.json`：

| 順序 | 類型 | 特徵 |
| --- | --- | --- |
| 1 | 普通怪 | 基礎攻擊，打倒給普通升級 |
| 2 | 普通怪 | 基礎攻擊，略強 |
| 3 | 精英怪 | 血量較高、攻擊較強，獎勵較好（見 Q-005） |
| 4 | 普通怪 | 過渡，回合略多 |
| 5 | Boss | 高血量、有週期性強攻擊 / 必殺技，擊敗即勝利 |

敵人類型行為：
- **普通怪**：固定攻擊力，每敵人回合造成一次傷害。
- **精英怪**：HP / 攻擊更高；獎勵更好（暫定提供保底高 rarity 三選一，見 Q-005）。
- **Boss**：HP 高；每隔固定回合數施放強攻擊（必殺技），平時為普通攻擊。

## 升級系統（Roguelite Build）

擊敗敵人後（Boss 除外，Boss 直接勝利）出現**三選一升級**，資料見 `Data/upgrades.json`。

升級類型（`target_type` / `effect_type`）：
- 提升某種 Peg 傷害（`target_type: peg`）
- 提升玩家最大 HP（`stat: max_hp`）
- 增加每回合球數（`stat: balls_per_round`）
- 解鎖 Blast Ball / Shield Ball（`unlock`）
- 提升補血效果（`peg: heal_peg`）
- 提升倍傷效果（`peg: double_peg`）
- 降低敵人攻擊（`stat: enemy_attack_down`）

規則：
- 三選一依 `rarity` 加權抽取，已解鎖 / 已達上限的選項排除（規則見 Q-004，待人類決議）。
- 套用後效果**持續整局**（不跨局，無存檔）。

## 勝敗條件（Win / Lose）

- **失敗（Game Over）**：玩家 HP ≤ 0。顯示死亡結算（到達層數、擊殺數、最終 build）。
- **勝利（Victory）**：擊敗第 5 場 Boss。顯示勝利結算（用時、剩餘 HP、最終 build）。
- 兩種結算都提供「重新開始 / 回主選單」。

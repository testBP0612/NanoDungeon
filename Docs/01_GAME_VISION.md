# 01 — GAME VISION 遊戲願景

## 一句話介紹

> 你是一團能量體，用彈珠台的方式駭入賽博空間的奈米地下城，每一發球都是一次精算過的攻擊，逐層擊潰病毒、防火牆與核心程式。

## 玩家幻想（Player Fantasy）

- 「我在用**運氣 × 規劃**打穿一座數位地下城。」
- 每回合屈指可數的能量球，要**算好角度**讓它在發光節點間彈跳，把傷害堆到最大。
- 透過**三選一升級**慢慢把自己養成一套組合技（build），看著傷害從個位數成長到爆發。
- 擊敗 Boss 的瞬間，是「我設計的 build 成功了」的成就感。

## 目標體驗（Target Experience）

- **節奏**：一局 5–10 分鐘，緊湊、可重玩。
- **手感**：彈珠彈跳、命中閃光、傷害數字跳出，要「爽、脆、亮」。
- **腦力**：瞄準有策略空間，升級有取捨，但門檻低、看一眼就懂。
- **風格**：霓虹賽博、暗底高光，像在操作一個發光的資料介面。
- **情緒曲線**：開局謹慎 → 中段 build 成形 → Boss 戰爆發 → 勝利結算釋放。

## 比賽展示價值（Why This Wins the Demo）

本專案真正要展示的不是遊戲本身，而是**前端工程師如何用 AI 建立可控的開發流水線**：

1. **可控（Controllable）**：人類訂方向，AI 不亂跑。核心方向只有人類能改。
2. **可驗收（Verifiable）**：每個 Phase 有 DoD，每次交付對照 `VALIDATION_CHECKLIST.md`。
3. **可迭代（Iterable）**：任務卡分段、資料驅動，改數值不改程式、加內容不重寫。
4. **可說故事**：現場能展示「Docs → 任務卡 → Codex 實作 → Review → Test → OPEN_QUESTIONS 回到人類」的完整閉環。

→ 評審看到的是一套**Self-Correcting Agent Workflow**：AI 會自我檢查、會把不確定交回人類，而不是黑箱式 Self-Improving。遊戲只是這套流程的「可玩證據」。

## 非目標（Non-Goals）

明確**不做**，避免範圍蔓延（scope creep）：

- ❌ 多人連線 / 對戰
- ❌ 真金投注 / 賭博機制
- ❌ 商城 / 內購 / 經濟系統
- ❌ 帳號登入 / 雲端存檔 / 本地存檔
- ❌ 複雜劇情、過場動畫、配音
- ❌ 複雜 3D 模型與大型動畫
- ❌ Web Export（只交付 Desktop）
- ❌ 關卡編輯器 / UGC
- ❌ 連鎖釘、連射球等高風險物理機制（MVP 階段排除）
- ❌ 追求精準數值平衡（只求 Demo 體驗穩定，見 `04_BALANCE_RULES.md`）

> 任何想超出非目標的需求，先寫進 `OPEN_QUESTIONS.md` 由人類決定，AI 不得自行加碼。
